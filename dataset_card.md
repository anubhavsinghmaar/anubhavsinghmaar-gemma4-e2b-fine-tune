# Dataset Card — Singhmaar Corp Fine-Tuning Dataset

## Overview

This dataset was created to fine-tune Gemma 4 E2B into a domain-specific internal AI assistant for A. Singhmaar Corp — a fictional mid-sized Indian conglomerate. The dataset is not scraped or generated by another AI. Every example was hand-crafted to reflect realistic enterprise interactions across five business divisions.

---

## Dataset Details

| Field | Value |
|---|---|
| Format | Alpaca-style JSON |
| Total examples | 100 |
| Train split | 90 examples |
| Eval split | 10 examples |
| Split seed | 42 |
| Language | English |
| License | Apache 2.0 |

---

## Schema

Each example is a JSON object with exactly three keys:

```json
{
  "instruction": "The user's question or request (1-2 sentences)",
  "input": "Optional supporting context such as an email, policy snippet, or numbers. Empty string if not needed.",
  "output": "A crisp, professional, company-aware answer (2-3 lines)"
}
```

## Design Principles

**Why 100 examples?**
The goal was to demonstrate domain adaptation, not instruction following at scale. 100 carefully written examples across 10 distinct domains is sufficient to shift the model's tone and knowledge grounding without overfitting to specific phrasings.

**Why Alpaca format?**
Alpaca format (instruction, input, output) is the most widely supported format for supervised fine-tuning across frameworks including TRL, Axolotl, and LLaMA-Factory. The optional `input` field allows including rich context like emails, policy excerpts, and meeting notes — which is realistic for enterprise use cases.

**What makes a good example?**
Each output was written to be concise (2-3 lines), professional in tone, and company-aware — meaning it references division names, portals, policies, and procedures specific to Singhmaar Corp. Generic answers that could apply to any company were rewritten to be specific.

```json
  "input": "",
  "output": "P1 incidents on SinghmaarTech's cloud infrastructure platform carry a 99.9% uptime SLA with a response time of 15 minutes and a resolution target of 4 hours. Breaches trigger automatic credit issuance as per the customer's service agreement."
}
```

---

## Limitations

- The company and all policies described are entirely fictional. Do not use this dataset to make claims about real organisations.
- 100 examples is a small dataset. The fine-tuned model will generalise to related queries within the same domain but may hallucinate on entirely new domains not covered in training.
- The dataset covers English only. Multilingual enterprise queries are out of scope.

---

## How to Extend This Dataset

If you want to build on this dataset for your own domain:

1. Follow the same Alpaca schema — instruction, input, output
2. Aim for at least 8 to 10 examples per domain you want to cover
3. Make outputs specific to your company — generic answers train a generic model
4. Include a mix of examples with empty input and examples with rich context in the input field
5. Vary the instruction phrasing — avoid having multiple examples that start with the same words
