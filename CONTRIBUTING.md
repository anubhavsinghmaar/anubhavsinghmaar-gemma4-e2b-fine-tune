# Contributing

This is a personal learning project, but contributions that improve it are welcome.

---

## What You Can Contribute

**More dataset examples**
The most useful contribution is adding more high-quality examples to the dataset. Follow the schema in `dataset_card.md` — instruction, input, output, Alpaca format. New domains like procurement, IT helpdesk, or operations are particularly useful.

**Bug fixes in the notebook**
If a cell breaks due to a library version update, a fix with a note explaining what changed is helpful.

**New domain adaptations**
If you use this structure to fine-tune on a different company or domain and want to share the dataset (with no real personal or sensitive data), open a PR with a note about what you changed.

---

## What to Avoid

- Do not add real company data, real employee names, or real policy documents
- Do not commit API tokens, HuggingFace credentials, or any secrets
- Do not add model weight files — those belong on HuggingFace, not in a GitHub repo

---

## How to Submit

1. Fork the repo
2. Make your changes on a new branch
3. Open a pull request with a short description of what you changed and why

There is no strict review process — this is a learning project, not a production system.
